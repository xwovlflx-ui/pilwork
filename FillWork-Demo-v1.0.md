# FillWork Field Demo v1.0

## 1. 목적과 범위

FillWork Demo v1.0은 실제 PostgreSQL이나 외부 API 없이 3명의 현장 기사와 1명의 관리자가 핵심 업무 흐름, 모바일 사용성, PWA 설치성, 오프라인 대응을 검증하기 위한 현장 테스트 빌드다.

- 데이터 소스: 브라우저 Mock Data와 `localStorage`
- 인증: Mock Token
- 파일/사진: 화면용 Mock 콘텐츠이며 실제 S3 업로드 없음
- 지원 화면: Dashboard, Project, Document, Wiki, Database, Calendar, Chat, File, Approval, CRM, Statistics, Notifications, AI Assistant, Settings, Admin
- 권장 테스트 기기: Android Chrome 2대 이상, iPhone Safari 1대, 관리자 PC Chrome 1대

## 2. 데모 계정

Mock 모드에서는 8자 이상 임의 문자열로 로그인한다. Database 모드는 로컬 Seed 환경변수로 지정한 비밀번호를 사용한다.

| 역할 | 이름 | 이메일 | 주요 테스트 |
|---|---|---|---|
| 조직 관리자 | 김서현 | `admin@fillwork.kr` | 관리자 센터, 승인, 통계, 전체 현황 |
| 현장 기사 1 | 김지연 | `field1@fillwork.kr` | 방문 일정, 프로젝트, 사진/파일 흐름 |
| 현장 기사 2 | 박민수 | `field2@fillwork.kr` | 채팅, 긴급 호출, 전자결재 흐름 |
| 현장 기사 3 | 이서연 | `field3@fillwork.kr` | 문서, 고객 확인, CRM 흐름 |

기존 역할 및 승인 테스트용 계정도 유지한다. 데모 화면에서 역할 분리 검증이 필요할 때 `super@fillwork.kr`, `manager@fillwork.kr`, `member@fillwork.kr`, `guest@fillwork.kr`, `pending@fillwork.kr`을 사용할 수 있다.

## 4. Mock Data 초기화

1. 화면 우측 하단 `현장 데모` 버튼을 연다.
2. `Mock Data 초기화`를 선택한다.
3. 확인 대화상자에서 초기화를 승인한다.
4. `fillwork.*` 형식의 브라우저 저장값이 삭제되고 기본 계정이 다시 생성된다.
5. 세션이 종료되고 로그인 화면으로 이동한다.

초기화는 해당 브라우저와 프로필에만 적용된다. 다른 기기의 Mock 상태에는 영향을 주지 않는다. 프로젝트·문서·채팅·파일의 화면 Mock은 코드에 고정된 읽기 전용 기준 데이터이므로 새로고침만으로 원래 상태를 유지한다.

## 5. PWA 설치 절차

### Android Chrome

1. HTTPS 또는 개발 환경의 `localhost`로 FillWork를 연다.
2. 한 번 새로고침해 Service Worker 등록을 완료한다.
3. `현장 데모` 패널에서 `이 기기에 설치`를 누른다.
4. 설치 버튼이 보이지 않으면 Chrome 메뉴의 `앱 설치` 또는 `홈 화면에 추가`를 사용한다.
5. 홈 화면의 FillWork 아이콘으로 실행하고 주소 표시줄이 없는 standalone 모드인지 확인한다.

### iPhone Safari

1. Safari에서 FillWork를 연다.
2. 공유 버튼을 누른다.
3. `홈 화면에 추가`를 선택한다.
4. 홈 화면 아이콘으로 실행한다.

### Desktop Chrome/Edge

주소창 설치 아이콘 또는 `현장 데모 > 이 기기에 설치`를 사용한다. 브라우저 정책상 설치 이벤트가 제공되지 않으면 패널에 `브라우저 메뉴에서 설치`로 표시된다.

## 6. 오프라인 동작 기준

- 방문한 HTML 화면과 정적 자산은 Runtime Cache에 저장한다.
- 네트워크가 끊기면 상단에 오프라인 상태 배너를 표시한다.
- 이전에 방문하지 않은 경로를 요청할 수 없으면 `/offline` 안내 화면을 표시한다.
- 인증과 Mock 데이터는 브라우저 `localStorage`를 사용하므로 같은 기기에서 유지된다.
- 실제 파일 업로드, 서버 동기화, 여러 기기 간 데이터 공유는 이 버전의 범위가 아니다.
- Service Worker 캐시는 버전 이름 `fillwork-demo-v1.0.1`로 관리하며 새 버전 활성화 시 이전 캐시를 제거한다.

## 7. 현장 테스트 사전 체크리스트

### 환경

- [ ] 테스트 URL이 HTTPS 또는 `localhost`로 제공된다.
- [ ] 4개 테스트 기기의 날짜와 시간이 자동 설정되어 있다.
- [ ] Android Chrome/Safari/관리자 PC Chrome 최신 안정 버전을 사용한다.
- [ ] 각 기기에서 기존 FillWork 저장 데이터와 설치 앱을 제거한 뒤 시작한다.
- [ ] Wi-Fi와 LTE/5G 전환이 가능하다.
- [ ] 스크린샷과 화면 녹화를 허용했다.

### 설치 및 로그인

- [ ] Manifest가 로드되고 FillWork 이름과 아이콘이 보인다.
- [ ] Android/PC에서 설치 프롬프트 또는 브라우저 설치 메뉴가 동작한다.
- [ ] iPhone에서 홈 화면 추가 후 standalone 실행이 가능하다.
- [ ] 4개 데모 계정이 Mock 모드에서 8자 이상 임의 문자열로 로그인된다.
- [ ] 기사는 `/admin` 접근 시 Dashboard로 돌아간다.
- [ ] 관리자는 `/admin`에 접근할 수 있다.
- [ ] 로그아웃 후 보호 화면으로 직접 이동하면 로그인 화면으로 전환된다.

### 메뉴 및 반응형

- [ ] Dashboard, Project, Document, Chat, File, Admin 핵심 화면이 열린다.
- [ ] 나머지 메뉴가 모두 404 없이 열린다.
- [ ] 390px에서 햄버거 메뉴가 열리고 현재 메뉴가 표시된다.
- [ ] 768px에서 콘텐츠가 잘리거나 가로로 겹치지 않는다.
- [ ] 1440px 관리자 화면의 3단 구성이 유지된다.
- [ ] 기기 Safe Area에서 버튼이 시스템 UI와 겹치지 않는다.

### 오프라인

- [ ] 온라인 상태에서 Dashboard, Project, Chat, File을 각각 한 번 방문한다.
- [ ] 비행기 모드 전환 후 오프라인 배너가 2초 내 나타난다.
- [ ] 방문했던 화면을 다시 열 수 있다.
- [ ] 방문하지 않은 화면은 오프라인 안내 화면으로 처리된다.
- [ ] 온라인 복귀 시 배너가 사라지고 새로고침이 성공한다.
- [ ] 오프라인 전환 전후 로그인 세션이 유지된다.

### 초기화

- [ ] 관리자가 가입 대기 사용자를 승인한다.
- [ ] `Mock Data 초기화` 실행 후 승인 내용이 원상 복구된다.
- [ ] 초기화 후 세션이 종료된다.
- [ ] 네 개 기본 데모 계정이 다시 로그인된다.
- [ ] 초기화 중 중복 클릭으로 오류가 발생하지 않는다.

## 11. 알려진 제한

- 화면 간 데이터는 실제로 동기화되지 않는다.
- 칸반 이동, 메시지 전송, 파일 업로드는 시각 데모 범위다.
- 오프라인 작업 큐와 충돌 해결은 실제 API/DB 연결 단계에서 구현한다.
- iOS는 `beforeinstallprompt`를 지원하지 않으므로 Safari 공유 메뉴를 사용한다.
- 브라우저가 저장소를 정리하거나 시크릿 모드를 종료하면 Mock 세션이 사라질 수 있다.
